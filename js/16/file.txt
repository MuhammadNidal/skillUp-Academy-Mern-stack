üß∞ 1. localStorage & sessionStorage

Both are part of the Web Storage API ‚Äî which lets you store data inside the browser.

They‚Äôre like small databases built right into your browser.

üß© Difference Between localStorage and sessionStorage
Feature	localStorage	sessionStorage
üïí Lifetime	Data stays forever (until you delete it manually or clear browser data)	Data is deleted when the tab or browser is closed
üåç Scope	Shared across all tabs and windows of the same site	Only for the current tab
üíæ Storage Limi5MBt	Around 	Around 5MB
üß† How to Use Them

Both have the same methods:

Method	What it does
.setItem(key, value)	Save data
.getItem(key)	Read data
.removeItem(key)	Delete specific data
.clear()	Delete all data
üß© Example 1 ‚Äî localStorage
// Save data in localStorage
localStorage.setItem('username', 'Nidal');

// Get data from localStorage
let user = localStorage.getItem('username');
console.log(user); // Output: Nidal

// Remove one item
localStorage.removeItem('username');

// Clear all data
localStorage.clear();


üìù Explanation:

setItem saves a key‚Äìvalue pair (both are strings).

getItem reads it.

removeItem deletes one key.

clear wipes everything.

üß© Example 2 ‚Äî sessionStorage
// Save data in sessionStorage
sessionStorage.setItem('theme', 'dark');

// Get data
let theme = sessionStorage.getItem('theme');
console.log(theme); // Output: dark

// Remove data
sessionStorage.removeItem('theme');

// Clear all
sessionStorage.clear();


üìù Explanation:

Works same as localStorage, but only for the current tab/session.

If you close the tab, data disappears.

‚ö†Ô∏è Important Note

localStorage and sessionStorage only store strings.

If you want to store an object or array, convert it to a JSON string using JSON.stringify().

üß© Example 3 ‚Äî Store an Object
let userInfo = {
  name: "Nidal",
  age: 22,
  country: "Pakistan"
};

// Save object (must convert to JSON string)
localStorage.setItem("user", JSON.stringify(userInfo));

// Get object (convert back to object)
let data = JSON.parse(localStorage.getItem("user"));
console.log(data.name); // Output: Nidal

‚úÖ Summary
Concept	localStorage	sessionStorage
Lifespan	Permanent	Until tab closed
Access	All tabs (same origin)	Only current tab
Use for	Long-term data (theme, login token)	Temporary data (form input)




2. Event Loop & Microtasks
üß† Simple Explanation:

JavaScript is single-threaded ‚Äî it runs one thing at a time.

But we can run asynchronous tasks (like setTimeout, API calls, etc.) without blocking other code.
The Event Loop helps manage:

Main stack (normal code)
Callback queue (like setTimeout)
Microtask queue (like Promises)

üß© Example:
console.log("Start");

setTimeout(() => {
  console.log("Timeout callback");
}, 0);

Promise.resolve().then(() => {
  console.log("Promise resolved");
});

console.log("End");

üßæ Output:
Start
End
Promise resolved
Timeout callback

üîç Explanation:

Start and End run first ‚Äî they are normal synchronous code.

Then the Promise (microtask) runs before the setTimeout (callback queue).

So event loop manages the order ‚Äî promises are prioritized.
