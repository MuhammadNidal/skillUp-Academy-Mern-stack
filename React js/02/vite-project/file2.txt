5. State in React
What is State?
State is like a memory inside a component.

It stores data that can change over time and affects how the component looks or behaves.

When state changes, React re-renders the component automatically.

useState Hook
In React functional components, we use the useState hook to create state.

const [variable, setVariable] = useState();
variable → current value of the state.

setVariable → function to update the state.

initialValue → default value for the state.

Example:

javascript
Copy
Edit
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(count + 1)}>Increase</button>
    </div>
  );
}
When you click the button, setCount() updates the state, and React re-renders the UI.

Handling Events
React uses camelCase for event handlers:

onClick

onChange

onSubmit, etc.

You can write:

<button onClick={handleClick}>Click Me</button>
or inline:

<button onClick={() => alert("Hello!")}>Click Me</button>
Example:


function Greeting() {
  const handleClick = () => {
    alert("Hello from React!");
  };

  return <button onClick={handleClick}>Say Hello</button>;
}
Two-Way Binding
Two-way binding means:

When a user types something, it updates the state.

When the state updates, the input field shows the new value.

Example:

function InputExample() {
  const [name, setName] = useState("");

  return (
    <div>
      <input 
        type="text" 
        value={name} 
        onChange={(e) => setName(e.target.value)} 
      />
      <p>Hello, {name}</p>
    </div>
  );
}
How it works:

value={name} links the input field with the state.

onChange updates the state whenever the user types.
