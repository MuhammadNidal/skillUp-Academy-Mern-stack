1ï¸âƒ£ What is Express.js and Why Use It?

Theory:

Express.js is a backend framework for Node.js.

It helps us build web servers and APIs easily.

Without Express, writing routes and handling requests in Node.js can be long and messy.

Express gives us shortcuts and tools for:

Handling routes (/home, /about, etc.)

Managing HTTP requests (GET, POST, etc.)

Sending responses (HTML, JSON, files)

Middleware (extra features like authentication, logging)

Lightweight and fast âš¡

Easy to learn for beginners

Widely used in the industry (MERN stack = MongoDB, Express, React, Node)

Simple Analogy:
Think of Node.js as a "raw kitchen." Express.js is the organized chef who arranges recipes, ingredients, and tools so cooking (coding) becomes easy.

1ï¸âƒ£ Example: Sending a Simple JSON Object

Instead of just sending text, we can send static data like a JSON object (like a database but hardcoded for now).

const express = require('express');
const app = express();

// Static data
const students = [
  { id: 1, name: "Ali", course: "Web Development" },
  { id: 2, name: "Sara", course: "Digital Marketing" },
  { id: 3, name: "Nidal", course: "MERN Stack" }
];

// Route to get all students
app.get('/students', (req, res) => {
  res.json(students);  // send JSON response
});

// Start server
app.listen(3000, () => {
  console.log("Server running on http://localhost:3000");
});

2ï¸âƒ£ Example: Sending Single Student by ID (Dynamic Route)

We can fetch one student using app.get() with a route parameter.

// Route to get single student by ID
app.get('/students/:id', (req, res) => {
  const studentId = parseInt(req.params.id);
  const student = students.find(s => s.id === studentId);

  if (student) {
    res.json(student);
  } else {
    res.status(404).send("Student not found");
  }
});
